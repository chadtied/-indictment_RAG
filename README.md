法律起訴書推薦系統
### ****整體系統架構***

![image.png](attachment:d53cf848-1907-4c81-917b-1e485b029aa6:image.png)

**1. 資料預處理 (Data Preprocessing)**

系統首先將原始案件資料進行**結構化處理**，擷取大標題、犯罪事實、證據、詐騙類型（如警察詐騙）及受害管道（如電話、網路）等關鍵欄位。

- **生成摘要：** 利用 **GPT-4o** 模型針對「犯罪事實」與「證據」進行摘要，確保保留犯罪核心邏輯與細節。
- **文字處理：** 使用 **結巴（Jieba）斷詞**，並透過人工觀察剔除冗言贅字與停用詞，優化後續分析的品質。

**2. 微調嵌入模型 (Fine-tune Embedding Model)**

為了確保模型能準確解讀法律文本並避免誤判，系統使用自行生成的訓練資料對預訓練模型（如 **bge-m3** 或 **jina-embeddings-v2-base-zh**）進行**微調（Fine-tune）**。此步驟能強化模型對語意邏輯的理解，將案件投射至正確的向量空間中,。

**3. 向量化與相似度分析**

這是系統識別相似案件的核心技術環節：

- **向量化：** 將微調後的模型把犯罪事實與證據文本轉換為**高維向量**，以保留文本間的語意與邏輯關係。
- **分群與比對：**
    - 利用 **HDBSCAN** 演算法進行向量分群，將高相似性的案件自動歸類。
    - 基於 **餘弦相似度（Cosine Similarity）** 進行案件間的逐一比對，計算當前案件與資料庫案件的匹配分數，找出最具參考價值的案例。

**4. 推薦結果生成**

系統根據相似度計算結果，為用戶生成推薦報告，內容包含：

- **相似犯罪事實與證據的摘要**，方便人員快速閱讀。
    - **推薦案件編號與標題**。
- **詐騙類型與受害管道的匹配情形**，確保推薦結果與目標案件的關鍵特徵相符,。

透過這套流程，系統能顯著縮短司法專業人員搜尋類案的時間，減少主觀判斷帶來的偏差，進而提升判決的公正性與一致性
